# ADR-002: Embedding 模型選擇

## 背景
系統需要將戰役戰術相關的非結構化文本轉換為向量表徵，以支援檢索與語義匹配。  
因此必須選擇合適的 Embedding 模型，兼顧語言支援、準確度與效能。

## 選項
- **OpenAI text-embedding-ada-002**：效能佳，但需付費，且對軍事專有名詞支援有限。  
- **Sentence-BERT (SBERT)**：開源模型，語義表徵能力強，但多語言支援有限。  
- **BGE-M3**：開源模型，支援多語言，適合軍事文本，社群活躍。

## 決策
選擇 **BGE-M3** 作為系統的 Embedding 模型。  
理由：  
- 支援多語言，適合軍事文本中常見的英文縮寫與中文描述。  
- 開源且社群活躍，維護成本低。  
- 在語義檢索任務上表現良好，適合戰術知識檢索。  

## 後果
- 系統能有效處理軍事文本中的專有名詞與多語言內容。  
- 部署簡單，無需額外付費。  
- 若未來需要更高效能或專業領域模型，可能需考慮商用 Embedding API。  

## 狀態
Accepted
